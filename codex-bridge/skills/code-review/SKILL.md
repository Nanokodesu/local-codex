# code-review

## Description
当用户要求审查代码、检查风险、查找 bug、分析 PR、Review、审查、找问题时使用。

## Triggers
- 审查
- review
- 代码检查
- 找 bug
- 找问题
- PR
- pull request
- 风险
- 安全问题

## Instructions
使用代码审查视角回答。优先列出真实问题、风险和行为回归，不要先写总结。

输出顺序：
1. 深度思考：简短说明审查范围和判断策略。
2. 任务列表：列出已检查和待检查项。
3. 执行反馈：每次读取文件或运行检查前，用自然语言说明目的；完成后说明发现。
4. 结果输出：按严重程度列出问题，必须包含文件路径、位置、影响和建议修复方式。

如果没有发现问题，明确说明未发现阻断问题，并指出剩余风险或未运行的检查。
