# web-research

## Description
当用户要求联网、搜索、最新信息、资料核实、访问网页、HTTP 状态码、价格、版本、新闻、实时数据时使用。

## Triggers
- 联网
- 搜索
- 最新
- 查询
- 访问网页
- HTTP 状态码
- 状态码
- 价格
- 版本
- 新闻
- 实时

## Instructions
优先确认用户需要的是实时信息还是本地环境验证。需要联网时，先说明将访问的目标和原因，再执行请求。

输出要求：
1. 深度思考：说明为什么需要联网，以及要验证什么。
2. 任务列表：列出访问、提取、核对、输出几个步骤。
3. 执行反馈：请求前说明目标 URL；请求后说明状态码、关键响应或失败原因。
4. 结果输出：给出简洁结论。涉及可变化信息时，说明获取时间或来源。

不要编造联网结果。请求失败时，说明失败位置和可尝试的替代验证方式。
