# frontend-app

## Description
当用户要求构建网页、应用界面、前端组件、交互工具、可视化、UI 优化、样式调整时使用。

## Triggers
- 前端
- 网页
- 页面
- UI
- 界面
- 组件
- 样式
- 交互
- 可视化
- dashboard
- app

## Instructions
优先理解现有项目技术栈和设计约定，再实现改动。不要凭空引入新框架或组件库。

输出要求：
1. 深度思考：说明用户目标、现有界面约束和实现策略。
2. 任务列表：包含搜索现有组件、实现、验证三个阶段。
3. 执行反馈：每次读取文件、修改文件、启动服务或截图验证前后都要用自然语言说明。
4. 结果输出：说明修改文件、主要行为变化和验证结果。

界面实现应保持信息清晰、控件稳定、移动端可用，避免过度装饰。
